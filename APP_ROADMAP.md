# Sawasdee App 開發路線圖（iOS 上架）

> 目標：把現有的 Sawasdee 網站，做成可上架 App Store 的 iOS App。
> 策略：**後端重用、前端重做、功能先行、外觀後套、合規一次到位。**
> 環境：Windows 開發 + 一支 iPhone + Apple Developer 個人帳號（$99/年）。

---

## 0. 技術選型（先定好，不要中途換）

| 項目 | 選擇 | 原因 |
|------|------|------|
| 前端框架 | **React Native + Expo** | Windows 可開發、一套程式碼之後可出 Android、雲端打包免 Mac |
| 後端 | **沿用現有 FastAPI**（不重寫）| 最大加速器 |
| 手機測試 | **Expo Go**（開發）→ **TestFlight**（正式測）| 不用 Mac |
| 打包上傳 | **EAS Build / EAS Submit**（Expo 雲端）| 免買 Mac |
| 金流 | **Apple 內購（IAP / StoreKit）** | App Store 數位訂閱強制 |
| 管理後台 | **網頁**（沿用 FastAPI + HTML）| 用瀏覽器管理，不用做 App |

---

## 1. 頁面骨架（先定結構，外觀之後再說）

底部 5 個分頁（Tab Navigation）：

```
[探索]  [組局]  [動態]  [訊息]  [我的]
  │       │       │       │       │
探索列表  組局列表 動態牆  對話列表  個人檔案
  │       │       │       │       │
個人頁   組局詳情 貼文詳情 聊天室   設定
聊天室            留言             訂閱(Premium)
```

獨立頁（不在 Tab）：登入、註冊、訂閱付款、聊天室。

**先把「有哪些頁 + 怎麼跳轉」接通，畫面陽春沒關係。**

## 3. 外觀套用（功能能動之後才做）

- 寫程式時把 **邏輯** 和 **樣式(StyleSheet)** 分開，改外觀不動功能。

- 一頁一頁美化，邊改邊用 Expo Go 在手機上看。

## 4. 合規硬條件（送審前必須全部完成）

> 這些是「進門門檻」，第一次送審就要有，不能事後補。

- [ ] **Apple 內購** 取代 Stripe（不可違規）
- [ ] **檢舉功能**（檢舉用戶 / 內容）
- [ ] **封鎖功能**（已有 block 模型，補 UI）
- [ ] **內容審核**（照片/動態 `is_approved`，初期人工審即可）
- [ ] **24 小時內處理回報** 的流程
- [ ] **隱私政策頁** + **服務條款 / EULA**
- [ ] **年齡分級 17+**
- [ ] 審核用 **測試帳號**（給 Apple 審查員登入）

  - [ ] 照片/動態審核佇列（通過 / 拒絕）
  - [ ] 用戶管理（停權 / 刪除）
  - [ ] 檢舉處理列表
  - [ ] 數據儀表板（註冊數、營收、在線）

---

## 6. 測試 → 上架

```
1. Expo Go 即時測（開發中，手機掃 QR）
2. eas build → eas submit → 上傳 TestFlight
3. TestFlight 找朋友用 iPhone 試用、收回饋、修 bug
4. App Store Connect 填上架資料（截圖、描述、隱私問卷）
5. Submit for Review → 過審 → Release 上架 🎉
```

---

## 7. 時程估計（你 + AI 輔助、每天投入、走 Expo）

| 階段 | 約略時間 |
|------|---------|
| P0 核心功能能動 | 3~5 週 |
| 套外觀 | 1~2 週 |
| 合規 + 後台 | 1~2 週 |
| TestFlight 測試修 bug | 1~2 週 |
| Apple 審核（可能來回）| 數天~2 週 |
| **合計到上架** | **約 1.5~3 個月** |
